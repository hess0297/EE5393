## CRN

The following reaction set extends the three-phase RGB methodology from the FIR filter handout to implement the biquad filter in Figure 2. It preserves the same core mechanisms while adapting to the added delay and multiple feedforward/feedback paths.

Key structure:
- Two delay chains: (R₁, G₁, B₁) and (R₂, G₂, B₂) store system state
- Scaling is implemented via successive halving (→ 1/8 gains)
- Output Y is formed from combined contributions of A, C, and E
- Feedforward/feedback paths are created through branching (e.g., B₁ → F + C + R₂)

Full CRN vs simulation:
- Groups 1–2: define the actual signal processing (transfer, delay, scaling)
- Groups 3–5: provide indicator species, phase control, and acceleration

In this simulation:
- Phase sequencing (blue→red→green→blue) is enforced directly in code
- Groups 3–5 are omitted because they only implement control/acceleration
- Including them in SSA introduces unnecessary fast reactions (“churning”)

Result:
- Simulation uses only Groups 1–2
- Preserves correct filter behavior
- Improves efficiency and clarity
